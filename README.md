# 실습 교육 웹사이트 (Training Platform)

Single-course, self-hostable practice-training web app for students. Built with
Next.js 15 (App Router) + TypeScript, Prisma (Postgres/Supabase), Tailwind CSS,
and iron-session cookies. Files are stored in Supabase Storage. Runs locally with
`next start` (Node) or deploys to Cloudflare Workers via the OpenNext adapter.

Students log in with an entry code, work through sequentially-locked steps, and
upload a photo per step (upload auto-advances to the next step). Admins log in
with a shared code + name to view a student roster, manage entry codes, and edit
step content under a pessimistic per-step edit lock.

## Requirements

- Node 20+
- npm
- Supabase 프로젝트 (Postgres + Storage) — 커넥션 문자열과 `uploads` 버킷 필요

## Setup & Run

```bash
# 1. Install dependencies (also runs `prisma generate`)
npm install

# 2. Create your .env from the template, then edit the values
cp .env.example .env
#    - ADMIN_CODE: the shared admin login code
#    - SESSION_SECRET: a random string, at least 32 characters
#    - DATABASE_URL: Supabase Supavisor 풀러 (포트 6543, ?pgbouncer=true)
#    - DIRECT_URL: Supabase 직접연결 (포트 5432, migrate 전용)
#    - SUPABASE_URL / SUPABASE_SERVICE_ROLE_KEY: Storage 접근용 (서버 전용)

# 3. Create the database schema (uses DIRECT_URL)
npm run db:migrate

# 4. Seed Course(id=1) + 3 sample steps
npm run db:seed

# 5. Start the dev server
npm run dev
```

Open http://localhost:3000 — choose **학생 입장** (student) or **관리자 입장** (admin).

- Student entry codes are generated by an admin under **참가 코드**.
- Admin login uses the `ADMIN_CODE` from `.env` plus any name you type.

## Production (로컬 Node)

```bash
npm run build   # prisma generate && next build
npm start
```

## Cloudflare 배포

OpenNext 어댑터로 Cloudflare Workers에 배포한다. 환경변수는 `.env`가 아니라
`wrangler secret`으로 등록한다.

```bash
# 1. Cloudflare 로그인
wrangler login

# 2. 런타임 시크릿 등록 (DIRECT_URL은 런타임에 불필요 — 로컬 migrate 전용)
wrangler secret put ADMIN_CODE
wrangler secret put SESSION_SECRET
wrangler secret put DATABASE_URL
wrangler secret put SUPABASE_URL
wrangler secret put SUPABASE_SERVICE_ROLE_KEY

# 3. 로컬 프리뷰용 시크릿 파일 (wrangler secret과 별개, .env와 같은 값으로 채우기)
#    .dev.vars는 gitignore 대상 — ADMIN_CODE / SESSION_SECRET / DATABASE_URL /
#    SUPABASE_URL / SUPABASE_SERVICE_ROLE_KEY 를 KEY=value 형식으로 작성

# 4. 빌드 후 Workers 런타임에서 로컬 프리뷰
npm run cf:build
npm run cf:preview

# 5. 프로덕션 배포
npm run cf:deploy
```

## Notes

- Uploaded photos and step images are stored in Supabase Storage (bucket
  `uploads`) and served via `/api/uploads/[...path]`, which validates the storage
  key (allow-listed subdir + UUID filename) behind a session gate.
- Sequential step locking is enforced server-side (a student cannot submit ahead
  of their current step).
- Admin **Reset** on a code force-logs-out the bound student and recycles the
  code back to unused.
- AI grading is intentionally out of scope — a step is complete once its photo
  upload succeeds.
