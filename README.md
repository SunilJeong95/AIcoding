# 실습 교육 웹사이트 (Training Platform)

Single-course, self-hostable practice-training web app for students. Built with
Next.js 15 (App Router) + TypeScript, Prisma (SQLite), Tailwind CSS, and
iron-session cookies. No external services — one process, one SQLite file, one
`uploads/` directory.

Students log in with an entry code, work through sequentially-locked steps, and
upload a photo per step (upload auto-advances to the next step). Admins log in
with a shared code + name to view a student roster, manage entry codes, and edit
step content under a pessimistic per-step edit lock.

## Requirements

- Node 20+
- npm

## Setup & Run

```bash
# 1. Install dependencies (also runs `prisma generate`)
npm install

# 2. Create your .env from the template, then edit the values
cp .env.example .env
#    - ADMIN_CODE: the shared admin login code
#    - SESSION_SECRET: a random string, at least 32 characters
#    - DATABASE_URL: leave as file:./dev.db for local SQLite

# 3. Create the database schema
npm run db:migrate

# 4. Seed Course(id=1) + 3 sample steps
npm run db:seed

# 5. Start the dev server
npm run dev
```

Open http://localhost:3000 — choose **학생 입장** (student) or **관리자 입장** (admin).

- Student entry codes are generated by an admin under **참가 코드**.
- Admin login uses the `ADMIN_CODE` from `.env` plus any name you type.

## Production

```bash
npm run build   # prisma generate && next build
npm start
```

## Notes

- Uploaded photos and step images are written to `uploads/` (gitignored) and
  served via `/api/uploads/[...path]` with a path-traversal guard.
- Sequential step locking is enforced server-side (a student cannot submit ahead
  of their current step).
- Admin **Reset** on a code force-logs-out the bound student and recycles the
  code back to unused.
- AI grading is intentionally out of scope — a step is complete once its photo
  upload succeeds.
